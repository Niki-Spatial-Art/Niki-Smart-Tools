# A 股动作卡实验室

目标：先验证纪律是否提高胜率，再考虑更复杂的自动化。

## 三层结构

### 1. 回测层

把每天的“今日动作卡”当作策略信号，回看后续 1 个、2 个、3 个交易日是否达到：

- 入场区
- 第一止盈
- 第二止盈
- 止损
- T+1 后是否仍有卖出机会

先不追求复杂因子，先验证一句话：系统给出的动作卡，是否比临场手感更稳。

### 2. 仿真层

只做纸面交易，不自动下单。

每天运行报告后，把动作卡导出到 `data/paper_trade_journal.csv`。如果你真的买了，也先把它当成一笔仿真交易记录下来：

- 为什么买
- 买多少
- 买入价
- T+1 下一交易日怎么处理
- 是否按计划卖出

### 3. 审计层

每笔交易必须能回答三个问题：

- 买入前的理由是什么？
- 风控门有没有打开？
- 下一交易日卖出计划是什么？

没有这三项，就算涨了也不算合格交易。

## 当前原则

- 不直接照搬外部项目代码，只借鉴结构。
- 备用源字段不完整的数据，只能观察，不能进入可操作池。
- A 股短线默认 T+1，今天买入当天不能卖。
- 先跑 A 股和 ETF，等执行稳定后再扩展港股、美股和期权。

## 常用命令

从最新报告导出动作卡：

```powershell
python tools/action_audit.py export-plan --report reports/latest.json --journal data/paper_trade_journal.csv
```

查看已完成交易统计：

```powershell
python tools/action_audit.py summarize --journal data/paper_trade_journal.csv
```
