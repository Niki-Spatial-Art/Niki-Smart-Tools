# 星耀接口利用审计清单

本清单用于逐项核对星耀 AmazingData 开发手册目录是否进入工作台。状态分为：

- `已规划`：已经映射到工作台模块，但尚未完全实现。
- `需联调`：需要星耀账号/SDK/权限确认字段和返回样例。
- `候补`：不是当前优先级，但保留到后续扩展。
- `不实时`：不得进入 10 秒级实时刷新，只能日频/事件频。

## 核对原则

1. 真正高频刷新的只有 `3.5.3 实时行情数据` 和极少量 L0/L1 标的。
2. 基础数据、财务、股东、两融、龙虎榜、ETF份额、IOPV 都走日频/事件频。
3. 每个接口项必须有工作台落点，否则先标 `候补`，不盲接。
4. 每个数据进入动作卡前必须有 `source`、`timestamp`、`freshness`、`confidence`。
5. 没有完成联调的接口不得支撑高置信交易动作，只能显示为观察或计划。
