# WorkBuddy 评审说明：星耀工作台 v1

这是一份给 WorkBuddy / 星耀数智 skills / 其他 AI Agent 阅读的项目说明。目标是让外部助手理解当前工作台定位、星耀数据调用方式、iFind 配合方式，以及接下来应该优化哪里。

## 1. 项目定位

本项目不是普通聊天问答，也不是单纯数据接口 demo，而是一个本地 A股 / ETF / ETF期权投研工作台。

核心目标：

- 盘中看清楚：买不买、卖不卖、等不等。
- 盘后沉淀：为什么做、为什么不做、错过了什么、明天怎么修。
- 数据透明：每条关键数据都要有 source、timestamp、freshness、confidence。
- 人工确认：系统只生成动作卡、风险线和研究结论，不自动下单。

当前主屏想做成类似 Koyfin / Bloomberg Launchpad / FXReplay / BacktestHomie 的个人投研驾驶舱，而不是一堆接口标题和信息卡片。

## 2. 数据源分工

### 星耀 AmazingData / 星耀数智

定位：主数据源。

优先承接：

- 3.5.2 基础数据：代码表、交易日历、复权因子、证券基础信息。
- 3.5.3 实时行情：指数、股票、ETF、ETF期权快照，实时K线。
- 3.5.4 历史行情：历史快照、历史K线。
- 3.5.5 财务：资产负债表、现金流量表、利润表、业绩预告/快报。
- 3.5.6 股东股本：股东户数、股本结构、质押冻结、限售解禁。
- 3.5.7 股东权益：分红、配股。
- 3.5.8 两融：两融汇总、两融明细。
- 3.5.9 龙虎榜/大宗交易：资金热度和异动事件。
- 3.5.10 期权：期权基础资料、合约属性、ETF期权实时快照。
- 3.5.11 ETF：PCF、基金份额、NAV/IOPV、申赎数据。
- 3.5.12 指数：指数成分股、权重。
- 3.5.13 行业：行业成分、权重、行业日行情。
- 3.5.14 可转债、3.5.15 国债、3.6 金融算子：后续专题扩展。

星耀调用边界：

- 所有 AmazingData 接口调用前必须先登录。
- 项目统一通过 `monitor.xingyao_login()` 登录。
- 不允许业务代码绕过统一登录函数直接调用 SDK。
- 密码不写入仓库、README、报告、日志或聊天。

项目已兼容的环境变量：

```powershell
XINGYAO_ENABLED=true
XINGYAO_USER=your_username
XINGYAO_USERNAME=your_username
XINGYAO_PASSWORD=your_password
XINGYAO_HOST=101.230.159.234
XINGYAO_PORT=8600
XINGYAO_SDK_PATHS=C:\path\to\amazingdata_sdk;C:\path\to\xingyao_sdk
```

也兼容星耀数智 skills 文档里的别名：

```powershell
AD_USERNAME=your_username
AD_PASSWORD=your_password
AD_HOST=101.230.159.234
AD_PORT=8600
```

### iFind

定位：校准源，不再作为未来主源。

原因：

- 当前 iFind 仍有额度，但大约 2026-06-22 到期。
- 用户倾向不续费。
- 到期前应充分利用 iFind 做星耀/东财字段校验。

iFind 当前用途：

- 校验星耀和东财的实时价、涨跌幅、成交额、时间戳。
- 校验基础数据字段，比如 ST、停牌、行业、证券名称。
- 校验财务、公告、资金字段的可用性。
- iFind 返回与星耀/东财冲突时，工作台必须标注冲突，不输出高置信动作卡。

iFind 不应该承担：

- 长期主行情源。
- 全工作台强依赖。
- 到期后仍阻断系统运行。

### 东方财富

定位：星耀实时行情第一候补。

用途：

- 星耀实时快照或实时K线接不上时，东财补盘中价格。
- 用于观察池、全市场扫描、行业热度的候补。
- 不替代星耀的完整基础/财务/ETF/期权专题能力。

### 腾讯 / 新浪

定位：补充和应急。

用途：

- 腾讯：单票报价补充。
- 新浪：单票应急报价。
- 不能作为核心全市场实时源。

### 本地缓存

定位：最后兜底，只能观察。

规则：

- 如果星耀、东财、腾讯、新浪都失败，进入缓存观察模式。
- 缓存数据不能支撑高置信买入。
- 页面必须显示缓存时间和数据过期风险。

## 3. 工作台五层结构

### 第一层：实时层

刷新频率：10-60秒，或订阅式推送。

包含：

- 指数实时快照。
- 股票实时快照。
- ETF实时快照。
- ETF期权实时快照。
- 实时K线。

服务页面：

- 盘中主屏。
- 动作卡。
- ETF/A股观察池。
- ETF期权第一页。

决策问题：

- 买不买。
- 卖不卖。
- 等不等。
- 数据是否新鲜。

### 第二层：历史/回测层

刷新频率：日频或按需。

包含：

- 历史快照。
- 历史K线。
- 复权因子。
- 交易日历。
- 历史代码表。

服务页面：

- 回测复盘页。
- 历史研究库。
- 复权切换。
- 策略逐根回放。

### 第三层：基本面层

刷新频率：收盘后或公告后。

包含：

- 资产负债表。
- 现金流量表。
- 利润表。
- 业绩预告/快报。
- 股东户数。
- 股本结构。
- 股权质押/冻结。
- 限售解禁。

服务页面：

- A股基本面页。
- 财务风险卡。
- 股东风险卡。
- 长期过滤器。

规则：

- 不做 10 秒轮询。
- 买入前作为风险闸门。

### 第四层：热度/资金层

刷新频率：日内低频或收盘后。

包含：

- 两融汇总。
- 两融明细。
- 龙虎榜。
- 大宗交易。

服务页面：

- 资金热度页。
- 异动观察卡。
- 明日候选池。
- K线事件 marker。

规则：

- 只做资金证据。
- 不直接触发买入。

### 第五层：ETF/专题层

刷新频率：日频缓存，盘中引用。

包含：

- ETF PCF。
- 基金份额。
- NAV。
- IOPV。
- 指数成分/权重。
- 行业成分/权重/日行情。

服务页面：

- ETF折溢价页。
- QDII风险页。
- 行业轮动页。
- 指数结构页。

## 4. 星耀数智 skills 分工

星耀数智 skills 是 AI 编排层，不等于底层数据接口。

### 金融数据 skill

用途：

- 查询行情、财务、股东股本等金融数据。
- 给工作台提供自然语言查询能力。

接入建议：

- 工作台按钮：“问星耀金融数据”。
- 输出进入数据卡、动作卡、字段解释。

### 技术面指标 skill

用途：

- 计算超买超卖、趋势、能量、成交量、均线、路径等指标。

接入建议：

- 接入 Lightweight Charts 主图和副图。
- 给动作卡提供技术触发条件。
- 给回测页提供指标参数。

### 基本面指标 skill

用途：

- 盈利、成长、运营效率、盈余质量、安全性、公司治理、估值、股东、规模等指标。

接入建议：

- 做买入前基本面风险闸门。
- 做 A股长期过滤器。
- 做财务/股东风险卡。

### 因子分析 skill

用途：

- 因子检验、因子有效性分析、因子合成、拥挤度测算、分层回测、IC分析、回归分析和报告生成。

接入建议：

- 做候选池排序。
- 做历史研究库。
- 做盘后因子报告。
- 做策略评分，不直接下单。

## 5. 前端可视化方向

主图内核：

- 采用 `tradingview/lightweight-charts`。
- 用于 K线、成交量、MA、VWAP、IOPV、NAV、止损线、事件 marker、回测回放。

仪表盘层：

- 可参考 ChartKit。
- 用于 KPI、sparkline、gauge、heatmap、progress ring。

页面结构建议：

1. 盘中主屏：指数/股票/ETF/期权快照 + K线 + 动作卡。
2. ETF与行业结构：ETF折溢价、IOPV/NAV、行业热力图。
3. A股基本面与股东风险：财务、股东户数、质押、解禁。
4. 资金热度：两融、龙虎榜、大宗交易、明日候选池。
5. 历史研究库：回测、复权切换、逐根回放、复盘笔记。

## 6. 当前项目已实现的部分

已做：

- 项目定位从 iFind-first 改为 Xingyao-first。
- 新增星耀五层工作台视图。
- 新增星耀接口利用审计表。
- 新增星耀登录与数据流状态面板。
- 新增星耀数智 skills 层面板。
- 兼容 `XINGYAO_*` 和 `AD_*` 环境变量。
- `monitor.xingyao_login()` 统一执行 AmazingData 登录。
- 星耀探针可输出快照、K线、期权基础合约状态。
- 页面会显示星耀登录是否就绪、SDK路径是否找到、快照/K线/期权是否返回数据。

尚未完全实现：

- 真实 Lightweight Charts 图表接入。
- 星耀 3.5.2 到 3.6 每个接口的真实字段探针。
- ETF IOPV/NAV/PCF 可视化。
- ETF期权实时链、IV、Greeks、OI。
- 两融/龙虎榜/大宗交易事件打到 K 线上。
- 基本面风险卡自动生成。
- 因子分析 skill 与候选池排序联动。

## 7. 希望 WorkBuddy 帮忙优化的问题

请 WorkBuddy 从以下角度评审并给出优化建议：

1. 星耀 AmazingData 数据接口如何按 3.5.2 到 3.6 分批接入。
2. 哪些接口必须实时，哪些接口只需要日频或事件频。
3. 星耀数智四类 skills 如何嵌入工作台页面和动作卡。
4. iFind 在到期前如何最高效地校准星耀和东财。
5. iFind 到期后，哪些能力需要东财/腾讯/新浪/缓存补位。
6. ETF 折溢价、ETF期权、行业轮动、两融/龙虎榜应该优先做哪几个页面。
7. 如何避免工作台变成信息堆积，而是变成盘中决策驾驶舱。
8. 如何设计每个卡片的 source、timestamp、freshness、confidence。
9. 如何把技术指标、基本面指标和因子分析转成可复盘的动作卡。
10. 是否建议使用 WorkBuddy 作为外部 AI 助手，而不是替代本地工作台主屏。

## 8. 推荐的最终关系

```text
本地星耀工作台 = 主驾驶舱
├─ AmazingData / TGW = 登录、取数、缓存、探针
├─ 星耀数智 skills = 金融数据、技术指标、基本面、因子分析的 AI 编排
├─ iFind = 2026-06-22 前校准源
├─ 东方财富 = 星耀实时失败后的第一候补
├─ 腾讯 / 新浪 = 单票补充和应急
├─ Lightweight Charts = 金融主图内核
├─ ChartKit = KPI / heatmap / gauge / sparkline
└─ 百炼 / 通义 = 字段解释、复盘、知识库、报告生成
```

一句话总结：

本地工作台负责持续监控、图表联动、动作卡、风控和复盘；WorkBuddy / 星耀 skills 负责自然语言查询、指标计算、因子分析和解释增强。两者应该协作，不应该互相替代。
