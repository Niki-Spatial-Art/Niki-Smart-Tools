## 一、AmazingData 开发手册 新发现的能力

review 手册后发现 skill 文档未覆盖的关键能力：

### 3.6 金融算子（⭐ 重大发现）

**3.6.3 时间序列函数 (TimeSeriesFunction)**：可直接在 pandas DataFrame 上计算全套技术指标：
- `MA/EMA/SMA/WMA/DMA/AMA` — 六种均线
- `HHV/LLV/HHVBARS/LLVBARS` — 最高最低值
- `REF/REFX/SHIFT` — 引用算子
- `CROSS/LONGCROSS` — 交叉检测
- `UPNDAY/DOWNNDAY/NDAY` — 连涨连跌判断
- `COUNT/EVERY/EXIST` — 条件统计
- `SAR` — 抛物线转向
- `TR` — 真实波幅

**3.6.4 截面函数 (CrossSectionFunction)**：16个跨标的算子：
- `CSZSCORE` / `CSPCTRANK` / `CSRANK` — 截面排名
- `CSNORMALIZE` — 截面归一化
- `CSSTD` / `CSVAR` / `CSCORR` — 截面统计

**这意味着**：你不需要依赖外部技术指标 skill 来计算 K 线上的指标，星耀 SDK 本身就内置了完整的金融算子。技术面 skill 可以退居为"验证/解释"角色。

### 数据结构新发现

| 数据结构 | 核心字段 | 可视化价值 |
|---------|---------|-----------|
| **Snapshot** | 5档买卖盘 + IOPV + trading_phase_code | 实时盘口深度面板 |
| **SnapshotOption** | 总持仓量 + 合约类型 + 到期日 + 行权价 + 5档 | 期权链 + T型报价 |
| **SnapshotIndex** | 上证/深证分市场字段 | 指数仪表盘 |
| **trading_phase_code** | 'T'=连续竞价 'C'=集合竞价 'E'=闭市 | 实时交易状态灯 |
| **security_status** | 停牌/除权/ST/退市/融资融券标记 | 风险闸门前置过滤 |

## 三、逐区详细设计

### ② 中心K线大图（占据 60%+ 屏幕面积）

**内核**：TradingView lightweight-charts v5

**图层叠加顺序（从下到上）**：

| 图层 | 内容 | 数据源 | 刷新 |
|------|------|--------|------|
| L0 | K线蜡烛图 | 星耀 query_kline (1m/5m/日) | 订阅推送 |
| L1 | 成交量柱 | 同一K线数据 | 同K线 |
| L2 | MA5 (黄) / MA10 (蓝) / MA20 (紫) | TimeSeriesFunction.MA | 随K线更新 |
| L3 | VWAP 均价线 | TimeSeriesFunction 计算 | 盘中累积 |
| L4 | IOPV 参考线 (仅ETF) | Snapshot.iopv | 10s |
| L5 | 止损线 (虚线) | 动作卡计算 | 随动作卡更新 |
| L6 | 事件标记 (龙虎榜/两融/分红/解禁) | InfoData 事件接口 | 日频 | 

**交互功能**：
- 鼠标十字线联动所有面板
- 滚轮缩放时间轴
- 点击K线弹出当日细节卡
- 右键 → "问星耀技术面" → 调用技术面 skill
- 拖动选区 → 回测回放

**周期切换按钮栏**：
```
[1m] [5m] [15m] [30m] [60m] [日K] [周K] [回放模式]
```

### ③ 动作卡面板（右上，实时更新）

每张动作卡必须包含：

```
┌─────────────────────────────────┐
│ 🔴 买入信号        freshness: 3s │
│ 510300 沪深300ETF               │
│                                 │
│ 现价  4.818  ↑+1.41%            │
│ 触发  MA5金叉MA10 + RSI<30反弹  │
│ 建议  观察等待（尾盘确认）        │
│ 止损  4.705 (ATR×2)             │
│ 止盈  5.016 (前高)              │
│                                 │
│ ┌ source: xingyao              │
│ └ confidence: high ●           │
│     timestamp: 14:35:12        │
│     失效条件: MA5回穿MA10       │
└─────────────────────────────────┘
```

**动作卡类型与颜色**：

| 类型 | 边色 | 含义 | 触发条件示例 |
|------|------|------|------------|
| 🔴 BUY | 红 | 买入观察 | MA金叉 + RSI回升 + 放量 |
| 🟢 SELL | 绿 | 卖出/减仓 | MA死叉 + 溢价过高 + 止损触发 |
| 🟡 WAIT | 黄 | 等待/观望 | 数据过期、信号冲突、异常波动 |
| 🔵 INFO | 蓝 | 事件通知 | 分红公告、解禁提醒、龙虎榜上榜 |

**四个必带元数据**：

```json
{
  "source": "xingyao|eastmoney|tencent|sina|cache",
  "timestamp": "ISO8601",
  "freshness": "real_time(<60s)|recent(<5m)|delayed(<1h)|stale(>1d)|unknown",
  "confidence": "high|medium|low|none"
}
```

**规则**：
- confidence ≠ high → 动作卡标题旁显示 ⚠️，降级为"仅供观察"
- freshness = stale → 动作卡变灰，禁止输出买入
- source = cache → 红色横幅 "数据已过期"

**刷新策略**：每次快照轮询更新价格和 freshness，技术条件满足时触发新动作卡生成。

## 五、技术实现路线图（修正版）

基于手册 review 后的新发现，接入计划调整：

### Phase 0 追加项：金融算子探针

在 Phase 0（本周）加入：
- 测试 `TimeSeriesFunction.MA/EMA/CROSS/HHV/LLV` 在真实数据上的表现
- 测试 `CrossSectionFunction.CSZSCORE/CSPCTRANK` 在多标的数据上的表现
- 验证后，部分技术指标计算可以下沉到 SDK 算子层，不再依赖外部 skill

### 前端技术栈

| 组件 | 技术选型 | 原因 |
|------|---------|------|
| K线主图 | lightweight-charts v5 | 金融级性能，支持百万K线 |
| 仪表盘/热力图 | ChartKit 或 ECharts | 非K线的统计图表 |
| 工作台框架 | Streamlit 或 纯HTML+WS | 快速迭代 vs 性能 |
| 实时数据 | WebSocket (星耀订阅) 降级 HTTP轮询 | 减少带宽 |
| 本地缓存 | HDF5 (星耀推荐) | 回测和大数据场景 |
