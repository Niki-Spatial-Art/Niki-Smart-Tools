# 证据卡与成交归因闭环

工作台不把 AI 摘要、群消息、热榜或单日异动直接转换为交易。它们只能进入线索池；候选必须先完成证据卡，成交必须完成归因，才会成为可学习样本。

## 1. 候选证据卡

从模板创建本地文件：

```powershell
Copy-Item .\examples\research_evidence.example.json .\data\research_evidence.local.json
```

每张卡都必须记录：

- 原始来源和来源时间；优先公司公告、业绩材料、订单、行业价格或可核验的产业数据。
- 供需/景气逻辑，以及市场可能尚未充分定价的部分。
- 反例、触发条件和失效条件。
- 数据验证：价格、MA20、流动性、账户持仓/交易权限。
- 逻辑验证：市场状态、至少 2-3 个同产业链标的共振、非追高位置、催化或基本面证据。

只有数据与逻辑两层都通过，且账户与市场闸门开放时，仪表盘才标记为“提交人工复核”。这仍不是下单指令。

## 2. 成交归因

从模板创建本地文件：

```powershell
Copy-Item .\examples\trade_attributions.example.csv .\data\trade_attributions.local.csv
```

每笔本地确认成交，以 `trade_time` 和 `code` 对应一行归因。`primary_cause` 只能从以下类别选择：

```text
market | selection | entry | sizing | exit | discipline
```

这使得复盘能区分：市场环境不支持、产业判断错误、入场追高、仓位不当、卖点失误，还是没有遵守既定规则。未归因的成交不会被当作可用于扩大资金的有效样本。

## 3. 日常顺序

```text
账户快照 -> 持仓处理 -> 市场状态 -> 证据卡 -> 人工复核 -> 成交记录 -> 结果归因
```

月度收益目标只用于风险预算和复盘，不会打开新仓权限。工作台不连接券商，不自动下单。
