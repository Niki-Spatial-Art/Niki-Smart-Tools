## 工作台结构

复制 FFD 的结构，而不是依赖 FFD 的数据：

1. 情报层：今天发生了什么。
2. 映射层：影响哪些行业、概念、公司。
3. 行情层：市场是否确认方向。
4. 公式层：能否转成通达信条件。
5. 动作层：买、等、加、减、放弃，以及失效条件。

## 通达信优先工作流

### 盘前

输入：

- 隔夜新闻或用户贴的早报
- 昨日强势板块
- 当前持仓和候选股

输出：

- 今日三条主线
- 每条主线的前排观察池
- 不碰清单
- 09:25-09:40 只观察条件

### 盘中

输入：

- 通达信行情截图
- 板块涨幅、换手、量比、封单、盘口
- 用户仓位/可用资金

输出动作卡：

- 动作：买 / 等 / 加 / 减 / 放弃
- 触发：价格、分时、板块确认条件
- 失效：跌破、炸板、板块退潮、量能不足
- 仓位：几成或多少股
- 风险：T+1、回撤、追高风险

### 盘后

输入：

- 当日成交
- 持仓盈亏
- 强弱板块
- 买卖原因

输出：

- 今日是否遵守计划
- 哪个信号有效，哪个是噪音
- 公式/选股规则是否要调整
- 明天最多三件事

## 公式筛选主线

外部指标包不要一个个肉眼看。先批量静态筛：

| 分档 | 含义 | 动作 |
|---|---|---|
| A | 优先研究 | 转成 `XG:` 条件选股，做样本验证 |
| B | 可学习/可改造 | 提炼逻辑，不急着实盘 |
| C | 低优先级/条件受限 | 暂存，盘中不占注意力 |
| D | 未来函数/重绘/高风险 | 只学习，不用于实盘选股 |

筛选器已加入：

- `tools/tdx_formula_screen.py`
- `run_tdx_formula_screen.ps1`

默认导出目录：

```text
work/tdx_exports
```

运行：

```powershell
powershell -ExecutionPolicy Bypass -File ".\run_tdx_formula_screen.ps1"
```
