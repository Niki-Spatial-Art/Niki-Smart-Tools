# iFind 周额度使用计划

生成时间：2026-06-04
周期：按用户反馈，iFind 账号额度每周更新。

## 总原则

不要为了“用完额度”而刷数据。目标是把每周额度的 70%-85% 用在能改变动作卡的地方，并保留 15%-30% 给临时重跑、行情异常、系统调试和新主题扩展。

## 使用优先级

| 优先级 | 模块 | 原因 |
| ---: | --- | --- |
| 1 | 实时行情 | 直接影响盘中价格、买卖线、禁买/减仓卡 |
| 2 | 日内快照/高频数据 | 替代 Level2，做盘口强弱和复盘证据 |
| 3 | 公告查询 | 买入前风险闸门 |
| 4 | 智能选股 | 扩展候选池，但只做交叉验证 |
| 5 | 历史数据 | 回测必需，但要缓存，避免重复消耗 |
| 6 | 专题报表 | 盘后主题研究，不用于盘中追涨 |

## 盘中计划

| 时段 | 任务 | 数据 |
| --- | --- | --- |
| 09:10 | 盘前持仓和观察池 | 实时 + 历史缓存 |
| 09:35 | 开盘动作卡 | 实时 + 日内快照 |
| 10:00/10:30/11:00 | 半小时买卖动作卡 | 实时 + 少量快照 |
| 13:05/13:30/14:00/14:30 | 午后买卖动作卡 | 实时 + 快照 |
| 14:45 | 尾盘隔夜风险卡 | 实时 + 快照 + 公告 |

## 盘后计划

| 任务 | 数据 |
| --- | --- |
| 收盘复盘 | 历史 + 当日快照 |
| 强势观察回测 | 历史数据，优先增量缓存 |
| 主题池扩展 | 智能选股 + 专题报表 |
| 公告风险扫描 | 公告查询 |
| 学习报告 | 本地资料 + 主题证据 |

## 每日智能选股问句预算

建议每天使用 20-50 条，不必省到不用。

方向：

- AI 硬件主线回踩确认；
- PCB/覆铜板/高速材料；
- 电力设备超级周期；
- 机器人强势回踩；
- 首板/涨停扩散后的次日承接；
- 高位过热但未破趋势的观察池；
- 弱仓反弹减压候选。

## 历史数据保护

历史数据已使用约 35%，是最需要管理的部分：

- 新增标的拉 120 日；
- 旧标的尽量增量更新；
- 不做全市场重复历史拉取；
- 回测结果落盘复用；
- 盘中动作卡优先用实时和快照，不重复跑长历史。

## 动作卡接入

每次完整动作卡应显示：

- 数据来源：iFind 实时/历史/快照/公告；
- 数据时间；
- 是否可执行；
- 是否因风控降级；
- 是否需要用户补成交截图。

## 风控底线

额度充裕不等于可以扩大交易风险。日亏损软止损触发后，所有买入候选自动降级为观察，即使 iFind 智能选股给出强势票也不例外。
