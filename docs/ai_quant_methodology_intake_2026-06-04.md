# AI 量化方法论资料接入卡

生成时间：2026-06-04
来源：用户粘贴的三篇方法论文章：

- `让 AI 当你的量化分析师`
- `把交易直觉变成可计算的因子：5 层框架与 ATR 案例`
- `ultrawork：让 JS 当多 Agent 的指挥官`

用途：升级研究流程、因子验证、自动化编排边界；不直接生成买卖信号。

## 资料价值判断

| 资料 | 有用程度 | 接入层级 | 结论 |
| --- | --- | --- | --- |
| 让 AI 当你的量化分析师 | 高 | 研究工作流 | 应转化为“人定假设，AI 做实现和审计”的 SOP |
| 把交易直觉变成可计算的因子 | 高 | 因子研发 | 应转化为动作卡因子化模板 |
| ultrawork 多 Agent 指挥官 | 中 | 系统架构 | 对交易信号帮助小，但对自动化流水线编排有启发 |

## 核心吸收

### 1. 研究总监模式

系统不应该让 AI 自己“找一只会涨的股票”。正确分工是：

| 角色 | 负责什么 | 不负责什么 |
| --- | --- | --- |
| 用户 | 交易假设、风险偏好、是否执行 | 不需要手写全部代码 |
| AI | 写脚本、跑回测、找实现问题、生成报告 | 不替用户下单，不替用户承担风险 |
| 系统 | 保存假设、数据、动作卡、成交反馈、复盘 | 不用历史结果强行承诺未来收益 |

落地规则：每个新策略必须先有一句人话假设，再有可执行验证计划。

### 2. 交易直觉因子化

把盘中直觉翻译成因子时，至少过 5 层：

1. 直觉句子：例如“放量突破可能更强”。
2. 可计算定义：成交额、量比、ATR、均线距离、涨幅区间。
3. 信号形态：连续因子、0/1 事件、分档标签。
4. 验证指标：IC、胜率、事件后收益、最大回撤、样本数。
5. 动作卡规则：买/等/不买、触发价、止损线、仓位上限。

落地规则：动作卡不能只写“强势”，必须逐步补充“为什么强、强到能不能买、买错亏多少”。

### 3. 自动化编排边界

多 Agent 或脚本编排适合做：

- 刷新 iFind 数据；
- 生成 clean radar；
- 跑位置回测；
- 更新学习报告；
- 检查 JSON/CSV；
- 输出动作卡；
- 提醒用户确认成交。

不适合做：

- 自动登录券商；
- 自动下单；
- 为了月目标强制找买点；
- 在软止损后继续生成进攻型买入指令；
- 用模型观点覆盖账户风控。

## 对当前系统的具体升级建议

| 优先级 | 升级项 | 文件/模块 | 说明 |
| ---: | --- | --- | --- |
| 1 | 因子验证四件套 | `tools/ifind_position_backtest.py` | 增加事件胜率、事件后 1/2/5 日收益、样本数门槛 |
| 2 | 直觉到因子模板 | `docs/factor_research_playbook_2026-06-04.md` | 以后每个主观想法先变成因子定义 |
| 3 | 研究总监 SOP | `workflows/README.md` | 新策略按“假设-计划-实现-审计-动作卡”推进 |
| 4 | 自动化编排清单 | heartbeat 巡检 | 继续只做提醒和报告，不做自动交易 |

## 对短线操作的启示

1. 盘中看到强势票，先问它属于哪种信号：主题强、量价强、回测强，还是单纯涨得快。
2. 如果只能解释主题，不能解释买点，就放进观察卡，不放进买入卡。
3. 如果账户触发日亏损软止损，任何买入候选都自动降级为观察。
4. 成交后立即回填真实交易流水，下一轮动作卡才不会重复给旧仓建议。

## 结论

这三篇资料对系统有用，但用法不是“让 AI 更大胆买”，而是让系统更像一个研究台：

```text
直觉 -> 因子定义 -> 数据验证 -> 风控闸门 -> 动作卡 -> 成交反馈 -> 复盘样本
```

只有走完这条链，资料才会真正提高交易质量。
