## 借用文档：系统要吸收的规则

### 1. Regime Gate：先判断市场状态，再讨论因子

动作卡新增思路：

```text
市场状态 -> 因子适配 -> 仓位上限 -> 动作候选
```

落地规则：

- 趋势 Regime：允许动量、突破、强趋势观察，但仍需回撤止损。
- 震荡 Regime：突破信号降权，均值回归和区间交易优先。
- 急跌 Regime：所有买入候选默认降级为观察，除非是明确防守资产并有止损。
- 流动性不足 Regime：降低仓位和频率，优先减少碎单交易。

### 2. Portfolio Gate：选股不是终点，组合才是决策主体

任何买入候选必须同时回答：

| 检查项 | 必答问题 |
| --- | --- |
| 手续费 | 这笔交易金额是否足以覆盖最低佣金和卖出成本？ |
| 仓位 | 买入后单票/单 ETF 是否超出上限？ |
| 相关性 | 是否和当前持仓高度同向，导致重复押注？ |
| 行业集中 | 是否让同一主题过度集中？ |
| 止损风险 | 触发止损时，最大亏损是多少？ |
| 再平衡 | 是新增进攻，还是从弱仓位切到强仓位？ |

硬规则：

- 股票短线 `1万元以下` 默认禁止。
- 股票短线 `1万到2万元` 默认不做，只有 A 级强确认才允许例外。
- 常规股票短线试错尽量 `2万元` 起看。
- ETF 可小一些，但禁止把碎单做成高频来回。

### 3. Skill Decomposition：把系统拆成可审计模块

借用 CICC/Anthropic 的思路，但改成适合个人交易辅助的结构：

| Skill / Agent | 职责 | 禁止事项 |
| --- | --- | --- |
| A股策略 Skill | 大盘、风格、行业、题材状态判断 | 不直接给下单指令 |
| ETF 配置 Skill | ETF 估值、流动性、跟踪主题、再平衡建议 | 不忽略止损和手续费 |
| 港股科技 Skill | 恒生科技、港股通互联网、半导体/电子/机器人细分观察 | 不把低估值当短线买点 |
| AI 产业链 Skill | CPO、PCB、光芯片、算力、液冷、MLCC、服务器链 | 不追社媒热票 |
| 风控 Skill | 仓位、止损、手续费、冷却期、禁止买回 | 不为收益目标放松纪律 |
| 复盘 Skill | 成交流水、截图、偏差归因、次日风险 | 不用事后理由美化错误交易 |
| 学习 Skill | 每日材料分级、吸收规则、待实验清单 | 不把外部观点直接写成买入信号 |

### 4. Options Seller Rule：期权卖方只允许先做仿真纪律

期权卖方材料给系统的硬规则：

- 禁止裸卖作为默认策略。
- 优先价差结构，最大亏损必须可计算。
- 亏损达到权利金 `2-3x` 时必须退出或进入人工复核。
- 保证金缓冲至少 `30%-50%`。
- 未经长期仿真验证，不进入实盘动作卡。

### 5. Supply-Chain Bottleneck Rule：紫苏叶框架只做研究筛选

可借用的问题清单：

1. 这个环节是否是产业链不可替代节点？
2. 产能是否有限，扩产周期是否长？
3. 下游需求是否已经能被订单、业绩或资本开支验证？
4. 市场是否还没有充分定价？
5. 是否存在社媒吹票、幸存者偏差或提前建仓唱多风险？

落地边界：

- 只进入主题研究池和观察池。
- 必须用公开数据验证订单、毛利、产能和客户集中度。
- 不因海外大 V 唱多而追高。
